# BirdCLEF+ 2026 EDA Insights

These artifacts were generated by `notebooks/01_data_eda.ipynb` on Kaggle and saved here as a lightweight EDA report snapshot. The raw competition audio and model outputs remain excluded from Git.

## Dataset Snapshot

- The training metadata contains **35,549 recordings** across **206 primary labels**.
- All **35,549 audio file paths were found** during the Kaggle run; `missing_audio_files.csv` only contains the header.
- The taxonomy file contains **234 labels**, including all **206 training labels** plus **28 taxonomy labels not present in train**.
- The soundscape label table contains **1,478 rows**, which should be treated as a separate, more evaluation-like domain.
- The sample submission contains **3 rows**, consistent with Kaggle's hidden-test workflow.

## Class Imbalance

![Class imbalance diagnostics](class_imbalance_diagnostics.png)

The dataset is strongly long-tailed:

- Mean recordings per class: **172.6**
- Median recordings per class: **125**
- Minimum recordings per class: **1**
- Maximum recordings per class: **499**
- The top **10** labels account for **13.9%** of all recordings.
- The top **30** labels account for **40.3%** of all recordings.
- **27** classes have **10 or fewer** recordings.
- **40** classes have **25 or fewer** recordings.

This suggests the baseline should not rely on raw random sampling forever. After the first clean run, the next training iteration should consider class-balanced sampling, rare-class augmentation, per-class validation metrics, or a loss that reduces dominance from frequent classes.

## Soundscape Labels

The soundscape annotations are often multi-label strings, such as:

- `23158;24279;24321;517063;555146;65380;66971`
- `24321;555146;65380;66971`
- `517063`

These rows represent a different acoustic setting from clean training clips: overlapping calls, background noise, and sparse temporal annotations are more likely. Use them for domain diagnostics and validation design rather than mixing them blindly into the clean training metadata.

## Spectrogram Observations

![Representative mel spectrograms](representative_mels.png)

The sampled spectrograms show several modeling-relevant patterns:

- Some examples contain repeated, high-energy phrases across much of the 5-second crop.
- Some calls are sparse and localized, meaning random crops may miss useful vocalizations.
- Low-frequency background energy and broad-band noise are visible in several examples.
- Different species occupy different frequency bands and temporal structures, supporting mel-spectrogram CNNs as a reasonable baseline.

These examples support using time/frequency masking, gain augmentation, and possibly multiple crops per recording during inference.

## Practical Modeling Takeaways

1. Start with the EfficientNet-B0 single-label baseline to establish a clean validation loop.
2. Track per-class metrics early; top-line accuracy can hide rare-class failures.
3. Add class-aware sampling or augmentation once the baseline is stable.
4. Keep secondary labels available for later multi-label or soft-label experiments.
5. Treat soundscape labels as a domain-shift diagnostic, not just more rows.
6. Preserve taxonomy metadata for family/genus-level error analysis.
